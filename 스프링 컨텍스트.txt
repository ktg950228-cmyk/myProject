ApplicationContext		:스프링 컨텍스트의 본체. 이 전체가 서블렛 컨텍스트 내부에 담기게 됨
 ├── BeanFactory		: 스프링 빈(Bean) 관리
 │     ├── BeanDefinition		: 어떤 객체를 만들지 정의
 │     ├── Bean			: 실제 생성된 객체
 │     ├── DI/Autowiring		: 의존성 주입 처리
 │     └── AOP Infrastructure	: 프록시, Advice, Pointcut, 외부 라이브러리인 Aspect있으면.
 ├── Environment	: 환경 관련 설정
 │     ├── PropertySources	: application.properties(★부트에서 초기세팅 작성하던 파일), YAML, env, JVM properties
 │     └── Profiles			: active profile(dev/test/prod 등)
 │
 ├── MessageSource	: 국제화(i18n) 메시지 관리
 │
 ├── ApplicationEventPublisher	: 이벤트 발행/리스닝 기능
 │
 └── ResourceLoader	: 외부 리소스 파일, 클래스패스, URL 등 로딩

Request 내용물

-Request Line : 요청 라인
	-메서드(ex. POST)
	-URL(get일땐 여기에 파라미터가 그냥 적혀있음. ?cPage=7 이런거)
	-HTTP버전(ex. HTTP/1.1)


-Header : Request메타정보 명시장소.
	-Host(요청대상 서버): example.com
	-Content-Type(바디 어떤타입인지):		application/json					ex): {"username":"Alice","age":25}
							 	application/x-www-form-urlencoded	ex): username=Alice&age=25
								multipart/form-data				ex): 이미지, 동영상
								text/plain						ex): "Hello"
								(자바가 받을때나 리액트의 axios가 자동 파싱할땐 여기 형식보고 정하는 듯)
	-Content-Length(본문 길이): 27
	-Authorization(인증 토큰): Bearer xxxxx 등


-CRLF(빈 줄) : 헤더-본문 구분 빈 줄. 헤더끝났단 소리


-Body(본문) : get에선 보통 비어있다. 여기에	json이 들어있으면 {"username":"Alice","age":25}같은 모양,
								파라미터가 들어있으면 username=Alice&age=25같은 모양으로 적혀있음
